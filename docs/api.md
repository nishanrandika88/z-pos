# API Specification

Z-POS primarily uses Supabase table APIs for CRUD and PostgreSQL RPC functions for transactional workflows.

## Auth

### Login

```ts
supabase.auth.signInWithPassword({ email, password })
```

Validation:

- `email`: valid email.
- `password`: required.

Errors:

- `401`: invalid credentials.
- `423`: account locked.
- `429`: rate limit exceeded.

### Forgot Password

```ts
supabase.auth.resetPasswordForEmail(email, { redirectTo })
```

## Catalog

### List Categories

`GET /rest/v1/categories?select=id,name,active,created_at,updated_at`

Response:

```json
[{ "id": "uuid", "name": "Juices", "active": true }]
```

### Create Item

`POST /rest/v1/items`

Request:

```json
{
  "item_code": "ORG-001",
  "barcode": "4790000000012",
  "item_name": "Orange Juice",
  "selling_price": 850,
  "category_id": "uuid",
  "availability": true,
  "active": true
}
```

Validation:

- `item_code`: unique per branch.
- `barcode`: unique per branch when present.
- `selling_price`: greater than zero.

## Discounts

### Create Discount

`POST /rest/v1/discounts`

Request:

```json
{
  "name": "Fresh Juice Promo",
  "percentage": 10,
  "applicable_type": "CATEGORY",
  "applicable_id": "uuid",
  "active": true
}
```

Constraint:

- one active discount per item/category target.

## POS

### Search Items

`GET /rest/v1/items?or=(item_code.ilike.*term*,barcode.eq.term,item_name.ilike.*term*)`

### Create Order

`POST /rpc/create_pos_order`

Request:

```json
{
  "order_payload": {
    "lines": [
      {
        "item": {
          "id": "uuid",
          "itemCode": "ORG-001",
          "itemName": "Orange Juice",
          "sellingPrice": 850
        },
        "quantity": 2,
        "automaticDiscount": 170,
        "lineTotal": 1530
      }
    ],
    "totals": {
      "subtotal": 1700,
      "automaticDiscount": 170,
      "manualDiscount": 0,
      "tax": 0,
      "grandTotal": 1530
    },
    "payment": {
      "method": "CASH",
      "amountTendered": 2000,
      "balanceReturned": 470
    }
  }
}
```

Response:

```json
"order-uuid"
```

Rules:

- Generate order number on server.
- Save order, order items, payment, and audit event in one transaction.
- Print only after this RPC succeeds.

## Reports

Recommended RPC/view endpoints:

- `daily_sales_report(start_date, end_date)`
- `monthly_sales_report(year)`
- `item_sales_report(start_date, end_date)`
- `category_sales_report(start_date, end_date)`
- `discount_usage_report(start_date, end_date)`
- `cashier_performance_report(start_date, end_date)`

Export formats:

- CSV: generated client-side for small reports.
- Excel/PDF: generated by Edge Function for production scale.

## Error Handling

Canonical shape:

```json
{
  "code": "VALIDATION_ERROR",
  "message": "Selling price must be greater than zero",
  "details": {}
}
```

Important codes:

- `UNAUTHORIZED`
- `FORBIDDEN`
- `VALIDATION_ERROR`
- `DUPLICATE_ITEM_CODE`
- `DUPLICATE_BARCODE`
- `ORDER_SYNC_CONFLICT`
- `PAYMENT_CAPTURE_FAILED`
